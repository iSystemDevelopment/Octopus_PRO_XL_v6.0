OCTOPUS PRO XL — HARDWARE DSP SOUND MODULE
Octopus PRO XL is a standalone ESP32-S3 dual-core performance instrument by DIODAC ELECTRONICS / iSystem. It combines a laser harp (8 beams), a 64-step groovebox, an 8-voice drum synthesizer, D-BEAM hand-proximity expression, and a laser show engine — all mixed through a shared insert-FX bus to stereo I2S audio out.

Use without a computer: SH1106 OLED + rotary encoder + SCALE + OC buttons give full access to sound design, pattern edit, song chains, and NVS save/load.
Use with this App: USB connects the module to Chrome or Edge. The App is a 1:1 parameter mirror over Web MIDI SysEx — same patches, grid, mixer, motion lanes, and user slots as the hardware menus. MIDI never passes through the web server; your browser talks directly to the device over USB.
Requirements: HTTPS origin (this site or localhost), SysEx permission, one browser tab at a time on Windows (only one tab can receive hardware echoes).

PLATFORM FEATURES
• 64-step sequencer — grid pages P1–P4 (16 steps each); live playhead on the SEQUENCER tab driven by STEP_SYNC.
• Song mode — 16 chain slots × up to 16 steps (bank A–D + repeats); 🔗 toggle + SONG chain editor.
• Seq arpeggiator — 8 patterns / 8 rates / gate; single-note steps expand through the active scale motif.
• Harp arpeggiator — 4 patterns / 4 rates; POLY8 / STRINGS / SOLO play modes; scale motif expansion.
• Shared room bus — one delay/reverb for all engines (MIXER → SHARED ROOM). SCENE recalls 16 room presets; LINK OFF (default) keeps FX-A insert recall from changing room character — only sends update.
• Organic Drive insert FX (FX-A index 12) — organic saturation for melody and drums.
• Drum engine — global D.PITCH, fog-reject gate, 4 kits (TR-909 / TR-808 / Trap / House), and a live beat-reactive drum scope (MIXER tab) that pulses on the actual pattern hits at the device playhead.
• 128 factory presets + 64 user sound slots per engine + 64 user pattern slots (header SLOTS).
• D-BEAM — local DSP only; route + target (harp or melody) in the D-BEAM panel; 5 response curves.
• Transport authority — hardware always owns play / stop / record / BPM; App buttons are read-only reflectors.
• App telemetry — CPU load in header; SYSEX MON panel (FILTER hides PING / STEP / CPU streams).
• USB-only MIDI — DIN MIDI removed; v6.0 SysEx protocol (manufacturer 0x7C device→App, 0x7D App→device).

CONNECTION & SYNC
Plug in USB — the App auto-connects to the ★ Octopus MIDI port, sends CMD_APP_SYNC_REQ, and imports the full device state. Badge shows Octopus ON once firmware replies. Heartbeat PING every ~800 ms keeps the OLED on "APP CONNECTED".
Hardware while connected: SCALE = play/stop · OC short = record-arm · encoder turn = BPM.
Playhead: switch to the SEQUENCER tab while playing — the cyan bar tracks the hardware step 1:1 (discrete, driven directly by STEP_SYNC, exactly like the device's 16-box bar). It never drifts or stutters when you move the mouse or the UI is busy. BPM range 40–240. ▶ / ■ / REC mirror device state only (clicking shows a hint). No inbound echo? Close other OctopusApp tabs, hard-refresh (Ctrl+Shift+R), open MON with FILTER off — you must see ← lines when moving hardware controls.

DRUM KITS
TR-909 / TR-808 / Trap / House — kit selection loads per-voice tuning (drum knobs follow). CC 70 on the drum MIDI channel switches kit live.

SAVE · LOAD · RESET · CLR
Popups mirror hardware scopes: Full / Banks+Pats / Motion / Settings. SAVE and Settings/Motion reset commit via NvsWorker then reboot; the App reloads and re-imports. Full/Banks reset arm an NVS flag and reboot in ~150 ms — the wipe runs on the next boot (same as OC+SCALE combo). LOAD reloads RAM with no reboot, then the App reloads. Only one persist action at a time. CLR = hardware Clear (active grid + P-locks + both sounds → preset 0). All destructive actions use YES/NO confirm.

USER SLOTS
Header SLOTS — 64 harp sounds, 64 seq sounds, 64 patterns. Save / load / rename; custom names persist via SysEx name blobs and echo on connect.

BANKS · CHAIN · GRID TOOLS
Banks A–D — four independent patterns; edits never bleed across banks.
🔗 Chain — song mode plays banks in order; up to 16 steps per song slot.
SONG — visual chain editor (bank + repeats per step). P1–P4 — pattern step pages (opens SEQUENCER grid).
CPY/PST — copy/paste active bank grid only. RND-H / RND-D — randomize melody or drum rows (density slider).

BEGINNER — WHAT IS THIS?
OctopusApp in MIDI Controller mode is a step sequencer in your browser. It sends normal MIDI (notes, knobs, program changes) to a synth, drum machine, or DAW. You do not need Octopus PRO XL hardware. Use Google Chrome (or Edge) at octopus.isystem.app — not Safari or Firefox.

QUICK START (3 MINUTES)
1. Open Chrome → octopus.isystem.app → click Allow when asked for MIDI.
2. Header MIDI menu → pick your output (anything without ★). Badge = MIDI OUT.
3. Tab INSTRUMENTS → melody Ch 1, drums Ch 10, set Program Change if needed.
4. Tab SEQUENCER → click grid cells (rows 1–8 tune, 9–16 drums) → set BPM → ▶ Play.
5. ■ Stop sends all notes off. Pattern auto-saves in this browser.

macOS + GOOGLE CHROME
Direct to hardware synth
• Plug in USB MIDI gear → Allow MIDI in Chrome → select device in App dropdown → match MIDI channel on synth → Play.

Into Logic / Ableton / Reaper / etc.
• Audio MIDI Setup → Window → Show MIDI Studio → open IAC Driver → tick Device is online (creates IAC Bus 1).
• OctopusApp MIDI port = IAC Driver Bus 1.
• DAW: new instrument track → MIDI input = IAC Bus 1 → channel 1 (melody) or 10 (drums).
• Program pattern here → Play → DAW receives MIDI. Arm + Record in DAW to capture.
• Optional: routing bar CLK sends MIDI clock (24 PPQN) while playing.

WINDOWS + GOOGLE CHROME
Direct to hardware
• Install USB interface driver if needed → Chrome → Allow MIDI → pick port in App → Play.

Into Ableton / FL Studio / Cubase / Reaper / etc.
• Install free loopMIDI (tobias-erichsen.de) → add a port (e.g. “OctopusApp”).
• OctopusApp MIDI dropdown = that loopMIDI port.
• DAW preferences → enable that port as MIDI input → instrument track → same port + channel.
• Play in OctopusApp — not inside the DAW transport (App owns tempo in MIDI mode).

USING WITH A DAW (IMPORTANT)
OctopusApp is not a VST plugin. You never “open” it inside the DAW — you route MIDI from Chrome into the DAW:
Chrome (this tab) → virtual cable (IAC / loopMIDI) or USB → DAW MIDI track → plugin or external out

• One App tab → one MIDI output port. For multiple devices, use the DAW to split channels.
• Melody grid rows → melody channel · Drum rows → drum channel (GM: kick 36, snare 38… editable in INSTRUMENTS).
• INSTRUMENTS CC knobs send live Control Change (volume, filter, etc.).
• EXP / IMP in routing bar = backup/restore full session JSON.

TWO MODES
• Octopus ON — ★ port + hardware SysEx. Full 3-tab editor, hardware transport.
• MIDI OUT — this guide. 2 tabs only. App play/stop/BPM. No SAVE/LOAD to device flash.

TABS IN MIDI MODE
• SEQUENCER — 64 steps (P1–P4), banks A–D, CPY/PST/RND/CLR, 🔗 song chain, factory melody/drum patterns.
• INSTRUMENTS — seq synth MIDI (Ch, PC, 8× CC, scope) + drum MIDI (notes per row, scope).

SONG MODE & PATTERNS (v6.2.03)
• 🔗 next to banks — song mode: plays chain rows in order (bank + repeats), same idea as hardware.
• PAT / EDIT — pattern grid ↔ chain editor. Pick Chain 01–16, set steps, bank per row, repeats ×.
• MELODY PATTERNS / DRUM PATTERNS dropdowns — load factory ROM into active bank (steps 1–16); MIDI fires on Play.
• Song chains + patterns save in browser / EXP JSON with the grid.

TROUBLESHOOTING
• No ports listed → Allow MIDI · refresh · replug USB · Windows: check driver.
• DAW silent → same virtual port on App + track input · matching MIDI channel · plugin armed.
• Stuck notes → press Stop in App (sends all notes off).
• Lost pattern → cleared browser data wipes localStorage — use EXP first.
