OCTOPUS PRO XL — HARDWARE DSP SOUND MODULE
Octopus PRO XL is a standalone ESP32-S3 dual-core performance instrument by DIODAC ELECTRONICS / iSystem. It combines a laser harp (8 beams), a 64-step groovebox, an 8-voice drum synthesizer, D-BEAM hand-proximity expression, and a laser show engine — all mixed through a shared insert-FX bus to stereo I2S audio out.

Use without a computer: SH1106 OLED + rotary encoder + SCALE + OC buttons give full access to sound design, pattern edit, song chains, and NVS save/load.
Use with this App: USB connects the module to Chrome or Edge. The App is a 1:1 parameter mirror over Web MIDI SysEx — same patches, grid, mixer, motion lanes, and user slots as the hardware menus. MIDI never passes through the web server; your browser talks directly to the device over USB.
Requirements: HTTPS origin (this site or localhost), SysEx permission, one browser tab at a time on Windows (only one tab can receive hardware echoes).

WHAT'S NEW IN v6.0
• 64-step sequencer — grid pages P1–P4 (16 steps each); live playhead on the SEQUENCER tab driven by STEP_SYNC.
• Song mode — 16 chain slots × up to 16 steps (bank A–D + repeats); 🔗 toggle + PAT/EDIT chain editor.
• Seq arpeggiator — 8 patterns / 8 rates / gate; single-note steps expand through the active scale motif.
• Harp arpeggiator — 4 patterns / 4 rates; POLY8 / STRINGS / SOLO play modes; scale motif expansion.
• OrgDrive insert FX (FX-A index 12) — organic saturation for melody and drums.
• Drum engine — global D.PITCH, fog-reject gate, 4 kits (TR-909 / TR-808 / Trap / House), live drum scope on MIXER + INSTRUMENTS tabs.
• 128 factory presets + 64 user sound slots per engine + 64 user pattern slots (header SLOTS).
• D-BEAM — local DSP only; route + target (harp or melody) in the D-BEAM panel; 5 response curves.
• Transport authority — hardware always owns play / stop / record / BPM; App buttons are read-only reflectors.
• App telemetry — CPU load in header; SYSEX MON panel (FILTER hides PING / STEP / CPU streams).
• USB-only MIDI — DIN MIDI removed; v6.0 SysEx protocol (manufacturer 0x7C device→App, 0x7D App→device).

CONNECTION & SYNC
Select the ★ Octopus MIDI port → click CONNECT (or auto-connect when detected) → App sends CMD_APP_SYNC_REQ → device echoes full state. Badge shows LINKING then ONLINE once firmware replies. Heartbeat PING every ~800 ms keeps the OLED on "APP CONNECTED".
Hardware while connected: SCALE = play/stop · OC short = record-arm · encoder = BPM · encoder long = SAVE menu.
Playhead: switch to the SEQUENCER tab while playing — cyan bar glides smoothly at 60 fps via a PLL slaved to STEP_SYNC (hardware anchors every step, App interpolates sub-step position). Snaps hard only on page change or pattern wrap. BPM range 40–240. ▶ / ■ / REC mirror device state only (clicking shows a hint). ▶
No inbound echo? Close other OctopusApp tabs, hard-refresh (Ctrl+Shift+R), open MON with FILTER off — you must see ← lines when moving hardware controls.

DRUM KITS
TR-909 / TR-808 / Trap / House — kit selection loads per-voice tuning (drum knobs follow). CC 70 on the drum MIDI channel switches kit live.

SAVE · LOAD · RESET · CLR
Popups mirror hardware scopes: Full / Banks+Pats / Motion / Settings. SAVE and scoped RESET reboot (device resumes in place). LOAD reloads RAM with no reboot. Only one persist action at a time — save, reset, and load block each other while in flight (same as firmware). CLR = hardware Clear (active grid + P-locks + both sounds → preset 0). SOFT RESET = working image only, no NVS, no reboot. All destructive actions use YES/NO confirm.

USER SLOTS
Header SLOTS — 64 harp sounds, 64 seq sounds, 64 patterns. Save / load / rename; custom names persist via SysEx name blobs and echo on connect.

BANKS · CHAIN · GRID TOOLS
Banks A–D — four independent patterns; edits never bleed across banks.
🔗 Chain — song mode plays banks in order; up to 16 steps per song slot.
CPY/PST — copy/paste active bank grid only. RND-H / RND-D — randomize melody or drum rows (density slider). Pages P1–P4 = steps 1–16 … 49–64.
