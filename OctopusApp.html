OCTOPUS PRO XL — HARDWARE DSP SOUND MODULE
Octopus PRO XL is a standalone ESP32-S3 dual-core performance instrument by DIODAC ELECTRONICS / iSystem. It combines a laser harp (8 beams), a 64-step groovebox, an 8-voice drum synthesizer, D-BEAM hand-proximity expression, and a laser show engine — all mixed through a shared insert-FX bus to stereo I2S audio out.

Use without a computer: SH1106 OLED + rotary encoder + SCALE + OC buttons give full access to sound design, pattern edit, song chains, and NVS save/load.
Use with this App: USB connects the module to Chrome or Edge. The App is a 1:1 parameter mirror over Web MIDI SysEx — same patches, grid, mixer, motion lanes, and user slots as the hardware menus. MIDI never passes through the web server; your browser talks directly to the device over USB.
Requirements: HTTPS origin (this site or localhost), SysEx permission, one browser tab at a time on Windows (only one tab can receive hardware echoes).

PLATFORM FEATURES
• 64-step sequencer — grid pages P1–P4 (16 steps each); live playhead on the SEQUENCER tab driven by STEP_SYNC.
• Song mode — 16 chain slots × up to 16 steps (bank A–D + repeats); 🔗 toggle + SONG chain editor.
• Seq arpeggiator — 8 patterns / 8 rates / gate; single-note steps expand through the active scale motif.
• Harp arpeggiator — 4 patterns / 4 rates; POLY8 / STRINGS / SOLO play modes; scale motif expansion.
• Shared room bus — one delay/reverb for all engines (MIXER → SHARED ROOM). SCENE recalls 16 room presets; LINK OFF (default) keeps FX-A insert recall from changing room character — only sends update.
• Organic Drive insert FX (FX-A index 12) — organic saturation for melody and drums.
• Drum engine — global D.PITCH, fog-reject gate, 24 kits (TR-909 / TR-808 / Trap / House + 20 App voicing presets), and a live UV energy-map drum scope (MIXER tab) that shows active voice energy at the device playhead.
• 128 factory presets + 64 user sound slots per engine on hardware (HARP/SEQ SYNTH → Save/Load Slot).
• D-BEAM — local DSP only; route + target in MIXER tab; enable/bypass in header D-BEAM button; live amplitude bargraph under expression controls (hardware telemetry).
• Transport — shared play/stop/rec state: hardware SCALE or App ▶/■ both toggle; device echoes keep buttons in sync.
• App telemetry — CPU load in header; SYSEX MON panel (FILTER hides PING / STEP / CPU streams).
• USB-only MIDI — DIN MIDI removed; v6.0 SysEx protocol (manufacturer 0x7C device→App, 0x7D App→device).

CONNECTION & SYNC
Plug in USB — auto-connect detects ★ Octopus port. On first connection in a browser session the App performs a hard-refresh (Ctrl+Shift+R equivalent) to ensure clean WebMIDI state, then reconnects automatically. After refresh: import state (orange SYNC), then green ON when playhead/transport mirror is live. On reconnect (replug or resync) the App sends APP_SYNC_REQ — wait for the burst to finish before editing; active steps, playhead, knobs, and patches should match hardware.
▶/■/REC work from App or hardware. Pressing ▶ with no connection shows the connect guide instead of failing silently.
Hardware while connected: SCALE = play/stop · OC short = record-arm · encoder turn = BPM.
Keyboard (DSP): 1 INSTRUMENTS · 2 MIXER · 3 SEQUENCER · Space play/stop · R record (auto-starts transport when arming).
Playhead: switch to the SEQUENCER tab while playing — the cyan bar tracks the hardware step 1:1 (discrete, driven directly by STEP_SYNC). It never drifts or stutters. BPM range 40–240.
INSTRUMENTS IDLE: when no ★ Octopus is connected, the INSTRUMENTS view shows the idle overlay. The panels activate automatically on connection.
No inbound echo? Close other OctopusApp tabs, hard-refresh (Ctrl+Shift+R), open MON with FILTER off — you must see ← lines when moving hardware controls.

DRUM KITS & VOICING
24 kits total — kits 0–3 (TR-909 / TR-808 / Trap / House) map to firmware synth voices; kits 4–23 (Jungle … Cosmic) are App voicing presets that use the same firmware kit with per-voice tuning, decay, volume and noise offsets. Kit selection loads per-voice tuning and the kit's character (drum knobs follow). CC 70 on the drum MIDI channel switches firmware kit live.
The classic voices are hand-tuned per kit: snare = body wavetable + bandpassed rattle + click (Tune = snap brightness, Noise = wires), clap = bandpass triple-burst with a noise layer (Tune = filter centre, Noise = layer), hats = 6-osc inharmonic metal (Tune = HPF cutoff, Noise = wash). Drm Pitch sweeps the whole bus; kick/toms/perc track it directly while snare body + hats normalise to the classic ×0.60 default.

SAVE · RESET · CLR
Header SAVE stores the workspace to SESSION 2–16 in browser JSON (no reload). SESSION 1 auto-saves. Optional checkbox mirrors to device NVS (reboots). Hardware factory RESET is menu-only — not in the App header. Header CLR clears the active P page (16 steps) and P-lock motion on the connected device; other P pages and sound presets are unchanged.

BANKS · CHAIN · GRID TOOLS
Banks A–D — four independent patterns; edits never bleed across banks.
🔗 Chain — song mode plays banks in order; up to 16 steps per song slot.
SONG — visual chain editor (bank + repeats per step). P1–P4 — 16-step pages within active bank (click to lock view while playing).
CPY/PST — copy/paste active P page (16 steps). CLR — clear active P page grid + P-locks on hardware (DSP mode) or motion lanes (MIDI mode). RND-S / RND-D — randomize synth or drum rows on active P page (density slider). Factory pattern dropdowns also load into the active P page.

OCTOPUS MIDI CONTROLLER · OCTOPUSAPP v6.6.01

WHAT IS THIS?
Octopus MIDI Controller — a free step sequencer in your browser. Standard MIDI out to hardware synths, drum machines, and DAWs. No Octopus PRO XL required. No server — Chrome talks USB MIDI directly.

WANT OCTOPUS DSP ENGINE?
Octopus PRO XL is a standalone Laser Harp & Groovebox. Plug ★ USB → Octopus DSP Engine shell (3 tabs, SysEx mirror) in this same app.

QUICK START
1. Unplug ★ Octopus (or pick a port without ★).
2. Chrome → octopus.isystem.app → Allow MIDI.
3. Badge = MIDI OUT · single pulpit page: instruments left (~67%) + grid right (~33%).
4. Paint cells · set BPM · ▶ Play. Route to hardware or virtual cable (IAC / loopMIDI) into your DAW.
Keyboard: Space play/stop · R record (auto-starts transport when arming motion lanes).
No port selected? ▶ Play opens a connect guide — pick any MIDI interface in the header, then play again.

PULPIT LAYOUT
• Utility header — LEN/TRN/OCT · ARP · JSON/MID/IMP · CLK + channels + MON.
• Sequencer bar — BANK, P1–P4, EDIT, SONG 01–16, SCALE, patterns, CPY/PST/CLR/RND.
• Left — SEQ SYNTH 24×CC (top) + drum note map + scope (bottom).
• Right — 16×16 matrix + playhead (isolated from DSP grid).

GRID TOOLS & MOTION
CPY/PST/CLR/RND affect the active P page only (16 steps within the bank). CLR also clears recorded motion (CC P-locks) on that page — purple dots on the grid disappear.
REC + ▶ — twist SEQ SYNTH CC knobs while playing to capture up to 4 motion lanes per bank (purple dot per step). Max 4 lanes — extra automation is dropped with a one-shot log warning.

SONG MODE
🔗 next to banks enables chain playback. Pick Chain 01–16 in toolbar. EDIT opens bank manager (bank + repeats per step). Saves in browser + JSON export.

SAVE / EXPORT
• Auto-save in browser (localStorage) — SESSION 1.
• JSON — full session (grid, chains, CC map, ARP, motion lanes).
• MID — Standard MIDI File of active bank pattern.
• IMP — JSON session or basic .mid → grid.

DAW ROUTING
macOS: IAC Driver Bus → DAW MIDI input. Windows: loopMIDI port. App owns tempo in MIDI mode — arm DAW track to record MIDI.
